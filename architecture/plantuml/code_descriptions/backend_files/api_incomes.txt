File: server/api/incomes.ts

Schema: `CreateIncomeSchema` (zod) — validiert Body für POST (userId, categoryId, source, amount, date, interval, note)

Exported handler: default async (event)

Funktionen / Verhalten:
- Handler(event):
  - `GET`: Validiert `userId` → ruft `IncomeService.getIncomesByUserId(userId)` → gibt Liste zurück
  - `POST`: Liest Body, validiert → baut `payload` → ruft `IncomeService.createIncome(payload)` → gibt Ergebnis zurück
  - Fehler/405 entsprechend

Aufruft / Abhängigkeiten:
- `IncomeService.getIncomesByUserId(Number(userId))`
- `IncomeService.createIncome(payload)`

Rückgabewerte:
- GET: Array von Income-Domain-Objekten (vom Service transformiert)
- POST: Objekt { message, income }
