File: server/api/transactions.ts

Exported handler: default async (event) via `defineEventHandler`

Funktionen / Verhalten:
- Handler(event):
  - Liest HTTP-Methode (`getMethod`) und Query-Parameter
  - Unterstützte Methode: `GET`
    - Validiert `userId` mit `QueryUserIdSchema`
    - Ruft `TransactionService.getAllTransactions(userId)` auf
    - Gibt das Ergebnis von `TransactionService.getAllTransactions` zurück (Array formatierter Transaktionen)
  - Nicht unterstützte Methoden → `405 Method not allowed`
  - Fehler: Loggt den Fehler und returned `createError` mit StatusCode

Aufruft / Abhängigkeiten:
- `TransactionService.getAllTransactions(Number(userId))`

Rückgabewerte:
- Bei Erfolg: Array von Transaktionen (im Format, das `TransactionService` liefert)
- Bei Fehler: HTTP-Error (Status 4xx/5xx)
