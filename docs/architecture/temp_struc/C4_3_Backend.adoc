== Controller (Kommunikation mit Frontend)

Controller: "Budgetcontroller" (nicht umgesetzt)

=== Controller: categories
    * Entgegennehmen der Anfrage vom Frontend
    * erhalten der Kategorien vom Service und ausgeben an das Frontend
    * erhalten der Kategorien-Liste vom Service und ausgeben an das Frontend
    * erhalten einer neuen Kategorie vom Frontend
    * senden der neuen Kategorie an den Service
    * Fehlermeldung an das Frontend senden

=== Controller: expenses
    * Entgegennehmen der Anfrage vom Frontend
    * erhalten der Ausgaben vom Service und ausgeben an das Frontend
    * erhalten der Ausgaben-Liste vom Service und ausgeben an das Frontend
    * erhalten einer neuen Ausgabe vom Frontend
    * senden der neuen Ausgabe an den Service
    * Fehlermeldung an das Frontend senden

=== Controller: goals
    * Entgegennehmen der Anfrage vom Frontend
    * erhalten der Challenge vom Service und ausgeben an das Frontend
    * erhalten der Challenge-Liste vom Service und ausgeben an das Frontend
    * erhalten eines neuen Challenge vom Frontend
    * senden des neuen Challenge an den Service
    * Fehlermeldung an das Frontend senden

=== Controller: incomes
    * Entgegennehmen der Anfrage vom Frontend
    * erhalten der Einnahmen vom Service und ausgeben an das Frontend
    * erhalten der Einnahmen-Liste vom Service und ausgeben an das Frontend
    * erhalten einer neuen Einnahme vom Frontend
    * senden der neuen Einnahme an den Service
    * Fehlermeldung an das Frontend senden

=== Controller: transaction
    * Entgegennehmen der Anfrage vom Frontend
    * erhalten der Transaktionen (Einahmen und Ausgaben) vom Service und ausgeben an das Frontend
    * erhalten der Transaktionen-Liste (Einahmen und Ausgaben) vom Service und ausgeben an das Frontend
    * erhalten einer neuen Transaktion (Einahmen und Ausgaben) vom Frontend
    * senden der neuen Transaktion (Einahmen und Ausgaben) an den Service
    * Fehlermeldung an das Frontend senden

Controller: "UserController" (nicht umgesetzt, kommt von erik)

== Service (Logik)

Service: BudgetService (nicht umgesetzt)

=== Service: CategoryService
    * erhalten der Kategorie vom Repository
    * erhalten der Kategorie-Liste vom Repository
    * senden der Kategorie an den Controller
    * senden der Kategorien-Liste an den Controller
    * erhalten einer neuen Kategorie vom Controller
    * senden der neuen Kategorie an das Repository
    * ID vom Controller erhalten
    * je nach wünsch löschen/aktualisieren (Inhalt der Kategorie mitgegeben) der Kategorie im Repository

=== Service: ExpenseService
    * erhalten der Ausgabe vom Repository
    * senden der Ausgabe an den Controller
    * erhalten einer neuen Ausgabe vom Controller
    * senden der neuen Ausgabe an das Repository
    * ID vom Controller erhalten
    * je nach wünsch löschen/aktualisieren der Ausgabe im Repository

=== Service: GoalService
    * erhalten der Challenge vom Repository
    * erhalten der Challenge-Liste vom Repository
    * senden der Challenge an den Controller
    * senden der Challenge-Liste an den Controller
    * erhalten eines neuen Challenge vom Controller
    * senden des neuen Challenge an das Repository
    * ID vom Controller erhalten
    * je nach wünsch löschen/aktualisieren (Inhalt des Challenge mitgegeben) der Challenge im Repository

=== Service: IncomeService
    * erhalten der Einnahme vom Repository
    * senden der Einnahme an den Controller
    * erhalten einer neuen Einnahme vom Controller
    * senden der neuen Einnahme an das Repository
    * ID vom Controller erhalten
    * je nach wünsch löschen/aktualisieren (Inhalt der Einnahme mitgegeben) der Einnahme im Repository

=== Service: TransactionService
    * Zusammenfassung von Einnahmen und Ausgaben von Service-Ebene (Ruft ExpenseService und IncomeService auf)

Service: UserService (nicht umgesetzt, kommt von erik)
